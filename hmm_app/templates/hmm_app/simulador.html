¿Qué estamos haciendo aquí?

Tienes un amigo que vive lejos. Cada día te dice qué hizo, pero tú nunca ves el clima donde él vive. El modelo intenta adivinar el clima a partir de las actividades.

Lo que SÍ ves (observaciones)
🚶 Caminar = 0
🛒 Compras = 1
🧹 Limpiar = 2
→
Lo que quieres saber (estados ocultos)
🌧️ Estado 0 = Lluvia
☀️ Estado 1 = Sol
{% csrf_token %} {% comment %} ── Secuencia de días ── {% endcomment %}

Paso 1 — ¿Qué hizo tu amigo cada día?

Escribe la secuencia de actividades separadas por coma.
Ejemplo: 0,1,2,0,0,1 = Caminó, fue de compras, limpió, caminó, caminó, fue de compras.

Longitud recomendada: entre 4 y 20 valores.
{% comment %} ── Estados ocultos ── {% endcomment %}

Paso 2 — Nombres de los estados ocultos (el clima)

Puedes cambiarlos a lo que quieras. Por defecto: Lluvia y Sol.

{% comment %} ── π distribución inicial ── {% endcomment %}

Paso 3 — ¿Con qué probabilidad empieza cada estado?

π (pi) = probabilidad del clima el primer día. Los dos valores deben sumar 1.0.
Ejemplo: 0.6 y 0.4 significa "60% de chance de que el día 1 sea lluvia".

{% comment %} ── Matriz A ── {% endcomment %}

Paso 4 — ¿Cómo cambia el clima de un día al otro? (Matriz A)

Matriz de transición A = probabilidad de que el clima de hoy sea X dado que ayer fue Y.
Cada fila debe sumar 1.0. Ejemplo: si hoy llueve, mañana hay 70% de chance de que siga lloviendo.

"{{ form_data.s1|default:'Sol' }}" cambia sigue igual
Fila 1 suma: a00 + a01 = 1.0  |  Fila 2 suma: a10 + a11 = 1.0
{% comment %} ── Matriz B ── {% endcomment %}

Paso 5 — ¿Qué hace tu amigo según el clima? (Matriz B)

Matriz de emisión B = probabilidad de cada actividad dado el clima del día.
Cada fila debe sumar 1.0. Ejemplo: cuando llueve, 60% de chance de que salga a caminar (bajo techo).
